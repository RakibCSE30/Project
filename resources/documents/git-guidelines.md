# Git & PR Guidelines

## Branch Naming Conventions
* `feature/` for new features (e.g., `feature/login-page`)
* `bugfix/` for bug fixes (e.g., `bugfix/fix-jwt-expiry`)
* `hotfix/` for critical production hotfixes

## Pull Request (PR) Workflow
1. Pull the latest code from `main`.
2. Create your feature branch.
3. Commit using conventional messages (e.g., `feat: add password reset button`).
4. Push to origin and open a PR against `main`.
5. Require at least 1 senior developer approval before merging.
